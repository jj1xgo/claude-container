# モデル運用ルールの明確化（Sonnet主体の徹底）

## Context

グローバル `~/.claude/CLAUDE.md`「4. モデルを使い分ける」は元々「Sonnet が常に主体、判断が必要な場面だけ上位モデルをサブエージェントとして呼ぶ」という設計だが、今回の長いセッションで実際にはユーザーが `/model` でセッション全体を Fable に手動切替し、以降 Plan Mode の計画立案・レビュー・一部実装を Fable 自身が直接担い続ける逸脱が起きた（ユーザーが「実装は Sonnet に」と言うまで Sonnet に手番が回らなかった）。

Fable サブエージェントにこの実態をレビューさせ、一次情報（ルール文面・実際の会話の流れ）で裏取りした結果、原因は偶発ではなく次の2点の構造的な欠落と判明した。

1. ルールは「Claude から `/model` 切替を提案しない」だけを定め、**ユーザー起点で `/model` 切替が起きた後にどう振る舞うか**が未定義だった
2. 「Plan Mode 中に上位モデル判断が必要になった場合、何を委譲し何を Sonnet が保持するか」の範囲が未定義だった（委譲すべきはトレードオフ比較・仕様解釈の局所のみで、Explore・計画ファイル執筆・ExitPlanMode は本来 Sonnet 保持のはずだが、その明記がなかった）

Fable はまた「サブエージェントの判断そのもの（トレードオフの当否）は事実確認で裏取りできない」という限界も指摘した（今回、Fable のレビュー指摘のうち事実に基づくものは裏取りできたが、"実務上問題なく回るか" という判断自体は検証不能だった）。ユーザーは、この限界を踏まえてもなお「常に Sonnet が主体、判断が必要な場面だけサブエージェント」の運用に統一することを望んでおり、破綻しうる唯一のケース（ユーザーが上位モデルと直接長い設計対話をしたい場合）には「復帰点宣言付きの一時切替」を例外として認める方針を確認済み。

## 変更内容

`~/.claude/CLAUDE.md`「4. モデルを使い分ける」節に、既存の3条件・運用ルールは変更せずに以下を追記する（編集前に既存慣行どおり `cp` でバックアップ）。

1. **`/model` 切替への対応を明記**: Claude からは提案しない（既存どおり）。ユーザーが明示的に `/model` でセッション全体を上位モデルへ切り替えた場合は例外として認めるが、切替直後に「どの判断が終わったら Sonnet に戻すか」を一言確認し、その役割が終わったタイミングで自ら Sonnet への復帰を申告する義務を負う
2. **「〜を Fable/Opus に確認して」の解釈を明記**: 既定では Agent ツールによるサブエージェント起動と解釈する（セッション全体の切替ではない）。ユーザーが `/model` を意図している場合は明示的にそう指示する
3. **Plan Mode 中の委譲範囲を明記**: 上位モデルへ委譲するのはトレードオフ比較・仕様解釈など判断の局所のみ。コードベース探索（Explore）・計画ファイルの執筆・ExitPlanMode の実行は Sonnet が保持する
4. **裏取り不能な判断の扱いを明記**: サブエージェントの出力のうち事実主張（ファイル存在・ログ等）は裏取りするが、トレードオフ判断そのものは一次情報が無く裏取りできない。対立する判断を Sonnet が独自に採否を決めず、選択肢としてユーザーに提示する

## 対象ファイル

- `~/.claude/CLAUDE.md`（git 管理外）のみ。3プロジェクト（claude-container・findsummits・sotlas-frontend）の CLAUDE.md はいずれも「モデルを使い分ける」節をグローバル参照のみで固有記述を持たないため変更不要（該当箇所を確認済み）。

## 検証

- 編集後、追記した4点が「4. モデルを使い分ける」節に矛盾なく収まっているか通し読みで確認
- ルール文言の変更のため実行時テストは無し。運用が守られているかは以降のセッションで確認する
